# 📣 План контента: как рассказать о боте

Чек-лист и готовые тексты под каждую площадку. Бери, адаптируй и публикуй.
Не публикуй всё в один день — раскидай по неделе, чтобы не выглядело как
автопостинг и чтобы каждая площадка дала свой охват.

---

## 🎬 Reels / TikTok / Shorts (45–60 секунд, вертикальное видео)

### Сценарий

| Сек | Что в кадре | Текст / закадр |
|---|---|---|
| 0–3 | Крупно: твоя физическая витрина (фото) → переход на чат с ботом | «Я джун. Заменил продавца-консультанта в нашем магазине нейросетью.» |
| 3–8 | Скринкаст Telegram. Печатаешь: «привет» | Бот отвечает дружелюбно, без заскриптованных «здравствуйте, я бот». |
| 8–18 | «сколько стоит iphone 17 pro max?» → ответ бота | Цена, наличие, без воды |
| 18–28 | «посоветуй смартфон до 25 тысяч» → бот выдаёт 1–2 варианта | На экране подпись: «Сам выбирает релевантные товары» |
| 28–38 | Быстрая склейка: VS Code с кодом, потом терминал с логами LM Studio | «Работает на локальной LLM. Ничего не уходит в OpenAI.» |
| 38–50 | Возврат к чату: ещё один вопрос — «есть кнопочный за 2 тыщи?» — ответ | «72 товара в каталоге. Знает только наш склад, не выдумывает.» |
| 50–60 | Конец: твоё лицо или эмодзи + ссылка | «Код на GitHub в био. Подписывайся, выкладываю всё, что строю.» |

### Хуки на первый кадр (выбери 1)

- «Я заменил продавца в нашем магазине нейросетью.»
- «Если ты джун и не знаешь что делать — построй вот это.»
- «Локальная LLM на ноутбуке + Telegram. Без OpenAI.»
- «За выходные собрал AI-консультанта для офлайн-магазина.»

### Технические советы по съёмке

- Используй встроенный **QuickTime → New Screen Recording** для скринкастов.
- Telegram — на iPhone в портретной ориентации. Запись через iOS «Запись экрана» (Control Center).
- Монтаж — **CapCut** (он у тебя установлен). Скорость 1.3–1.5x, чтобы влезть в минуту.
- Музыка — что-то спокойно-техничное из библиотеки CapCut, без слов.
- Подписи (субтитры) обязательно — 80% смотрят без звука.

### Лайфхак

Снимай **серию рилсов**, не один. Каждый — про одну фичу:
1. «Как бот понимает запросы на бюджет»
2. «Как сделал, чтобы бот не предлагал альтернативы»
3. «Почему локальная LLM, а не OpenAI»
4. «3 бага, на которые я попал»

Серия = алгоритм видит связь, охваты растут.

## 🐦 X (Twitter) — короткий тред

Формат: первый твит-хук, потом 4–6 коротких твитов с одним скрином в каждом.

```
Шипнул сегодня то, чем горжусь:

Telegram-бот-консультант для нашего семейного магазина техники.
Отвечает на вопросы покупателей на полностью локальной LLM
(Qwen 2.5 14B, M1 Pro 16 GB).

Без OpenAI, без подписок, без слива клиентских данных. 🧵
```

Дальше по одному твиту:

1. «Стек: Python + aiogram 3 + LM Studio. Каталог — обычный JSON на 70 SKU.» + скрин архитектурной схемы из README.
2. «Главный трюк: не пихать весь каталог в промпт. На запрос модель видит only top-3 релевантных + однострочники остальных. ~1500 токенов вместо 4000.»
3. «Узкое место — prompt processing 14B на 16GB. Решил настройками: ctx=4096, parallel=1. Сэкономил 3GB RAM.»
4. «Ловушка: openai-клиент по дефолту делает retries. LM Studio в этот момент сбрасывает текущую генерацию и начинает заново. Ответ не приходит никогда. Выключил max_retries=0.»
5. «Код открыт: github.com/<твой-юзер>/ai-bot. README на русском и английском. Если хочешь повторить у себя — все шаги внутри.»
6. «Я джун, ищу первую серьёзную работу. Если у тебя есть позиция или совет — пиши в личку.»

## 💼 LinkedIn — пост на 1500 знаков

```
За выходные собрал AI-консультанта для нашего семейного магазина техники.

Покупатель пишет в Telegram «сколько стоит iPhone 17 Pro Max» —
бот отвечает за 30–60 секунд. Цена, наличие, без воды.
«Посоветуй смартфон до 25 тысяч» — бот выдаёт 1–2 варианта строго
из нашего каталога, не выдумывает модели и цены.

Что важно — всё работает на локальной LLM (Qwen 2.5 14B через LM Studio
на MacBook M1). Никаких подписок на OpenAI, никакого слива клиентских
данных в облако.

Стек: Python 3.13, aiogram 3, OpenAI SDK (для совместимости с LM Studio),
дата-классы, JSON-каталог. Около 600 строк кода.

Из интересного:
— Не пихаю весь каталог в промпт. На каждый запрос ищу top-3 релевантных
  товара и подсовываю только их с деталями. Сэкономил половину токенов.
— Боролся с тем, что openai-клиент делает retries и сбивает генерацию
  на полпути. Поправил max_retries=0.
— Подобрал параметры LM Studio под 16GB RAM: контекст 4096,
  один параллельный слот. Освободил 3GB.

Код на GitHub: <ссылка>
README на русском и английском, можно повторить у себя.

Я fullstack junior, открыт к работе. Если ищете джуна
с упором на Python/AI — пишите в личку.

#python #aiengineering #telegrambot #localllm #junior
```

## 📚 Хабр — пост на 5–8 минут чтения

**Заголовок:** «Заменил продавца-консультанта в магазине техники локальной LLM на ноутбуке»

**Структура:**

1. **Введение** (300 слов). У нас семейный магазин. Покупатели пишут в Телеграм одинаковые вопросы 50 раз в день. Я джун, решил автоматизировать. Условия: бесплатно (никаких подписок OpenAI), приватно (клиентские данные не уходят), на одном ноутбуке.

2. **Архитектура** (схема + 200 слов). Telegram → aiogram → LM Studio. Почему JSON, а не Postgres. Почему aiogram, а не python-telegram-bot.

3. **Промпт-инжиниринг** (500 слов). История эволюции:
   - Сначала запихал весь каталог в системный промпт. Модель чихала, отвечала по 5 минут.
   - Сделал компактный однострочный каталог + динамическую вставку релевантных деталей. Полегчало в 3 раза.
   - Боролся с тем, что бот сливал внутренние ID типа `[phn-002]`. Поправил `__str__`.
   - Боролся с тем, что бот подсказывал альтернативы, когда не просили. Прописал в промпте.

4. **Грабли** (300 слов).
   - openai-клиент с дефолтными retries → LM Studio сбрасывает генерацию.
   - Модель грузится с контекстом 32K на 16GB → ноутбук на грани.
   - Кэш `@lru_cache` на промпте → правки не подхватываются без рестарта.

5. **Что дальше** (200 слов). RAG, картинки товаров, метрики, тесты, Docker.

6. **Код** — ссылка на GitHub.

Хабр любит честный рассказ от первого лица с граблями, а не туториал «10 шагов как сделать бота». Пиши как другу.

## 💡 На чём строится «личный бренд» джуна

1. **Регулярность важнее качества.** Лучше один не очень рилс в неделю, чем идеальный раз в полгода.
2. **Покажи путь, не результат.** Люди подписываются на людей, которые учатся, а не на тех, кто уже умеет.
3. **Конкретика бьёт абстракцию.** «Сэкономил 3GB RAM урезав контекст» в 10 раз сильнее, чем «оптимизировал производительность».
4. **Один проект — много контента.** Этот бот = 5 рилсов + 3 твит-треда + 2 поста + 1 статья. Не выгорай, придумывая новые проекты ради контента.
5. **Не стесняйся быть джуном.** «Я джун, делаю свой первый serious проект» — это сильный хук, не слабость.
